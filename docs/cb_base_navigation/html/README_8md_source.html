1 Constraint Based Base Navigation
2 ==================
3 [![Build Status](https://travis-ci.org/tue-robotics/cb_base_navigation.svg?branch=master)](https://travis-ci.org/tue-robotics/cb_base_navigation)
4 
5 Alternative to move_base (navigation stack ROS); a more simplistic base bavigation which works with the Costmap 2D ROS world model. In can handle goal position constraints and the goal orientation constraint is set up in a different way (see cb_base_navigation_msgs_srvs package).
6 
7 It provides two interfaces:
8 - Local Planner Interface: Interface to communicate with the local planner via actionlib. It works with local planners that adhere to the nav_core::base_local_planner interface.
9 - Global Planner Interface: Interface to communicate with the global planner, works with a new interface specified in cb_global_planner::GlobalPlannerPlugin. The following interaction is possible:
10  - Request a plan based on a set of position constraints
11  - Check if a plan is still valid
12
