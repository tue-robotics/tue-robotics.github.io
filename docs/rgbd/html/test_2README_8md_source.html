6 ## Test Nodes
7 
8 The nodes are split into two types, server and client nodes.
9 
10 The server nodes generate an artificial image and distribute it via their specific way of
11 communication. The client nodes use their specific way of communication to receive an image
12 and show it on a gui. The server and client nodes with matching names should always communicate with
13 each other.
14 
15 Server nodes:
16 
17 * `rgbd_test_server_ros`: Publishes an image via default ROS image message with the use of the
18 `ServerROS` class.
19 * `rgbd_test_server_rgbd`: Publishes an image via the custom RGBD message on a ROS topic with the
20 use of the `ServerRGBD` class. This also provides the RGBD service.
21 * `rgbd_test_server_shm`: Publishes an image via shared memory communication with the use of
22 the `ServerSHM` class.
23 * `rgbd_test_server`: Provides the interfaces of the `Server` class. This includes the interface of
24 `ServerRGBD` and `ServrSHM` combined with a hostname topic.
25 
26 Client nodes:
27 
28 * `rgbd_test_client_ros`: Subsribes to the default ROS image topics to receive an image with the
29 use of the `ClientROS` class.
30 * `rgbd_test_client_rgbd`: Subsribes to the custom RGBD topic with the
31 use of the `ClientRGBD` class.
32 * `rgbd_test_client_shm`: Receives an image via shared memory communication with the use of
33 the `ClientSHM` class. __Note__: This node has an initialization timeout. The shared memory needs
34 needs to be created before this timeout ends. So start the server node providing the shared memory
35 either before or directly after starting this node.
36 * `rgbd_test_client`: Receives images via either the RGBD topic or shared memory communication with
37 the use of the `Client` class. It switched between these two based on a hostname topic to determine
38 the availability of shared memory communication, but uses RGBD as a fallback.
39 
40 ## Test procedure
41 
42 As mentioned earlier, the server and client nodes with matching names should always communicate
43 with each other.
44 
45 The nodes communicating via ROS topics, `*_ros` and `*_rgbd`, can be started, closed and restarted
46 in any order. The nodes should always reconnect automatically and the client node should show new
47 images. Test this by opening the server before the client and vice-versa. Test the reconnection by
48 closing the first node and restart it again. Try mutltiple time intervals.
49 
50 The bare shared memory nodes, `rgbd_test_server_shm` and `rgbd_test_client_shm`, are not that
51 robust by themselves. These don't reconnect and the shared memory needs to be opened before the
52 timoeut of the client nodes has ended. Otherwise no communication will be established.
53 Test this by opening the server before the client or just directly after.
54 
55 The robust `rgbd_test_server` and `rgbd_test_client` should always reconnect. Apply the same test
56 procedure as the nodes using ROS topics. When both are running on the same machine, communication
57 should be happening via the shared memory. This can be checked by either `rostopic info` or
58 `rosnode info`.
59 
60 ### Cross testing
61 
62 To verify the robust behaviour of the classes perform the following tests:
63 
64 1. Combine the `rgbd_test_client` with `rgbd_test_server_rgbd`. This should always reconnect.
65 2. Extend the first test by starting a `rgbd_to_shm` node. (__Note__: This node should be run on
66 the same machine as the client node. This might require some remapping of the RGBD topic.) The
67 client node should switch from RGBD communication to shared memory communication. By closing the
68 `rgbd_to_shm` node, the client node should switch back to the RGBD topic. After both opening and
69 closing of the `rgbd_to_shm` node, check the way of communication again via either `rostopic info`
70 or `rosnode info`.
71 3. Start the `rgbd_test_server` followed by the `rgbd_test_client_shm`. Apply the same test as
72 used by the `rgbd_test_server_shm`. As `rgbd_test_server` provides the same shared memory
73 communication and extra RGBD topic and service.
74 4. In any test, the `rgbd_test_server` can be replaced by the combination of the
75 `rgbd_test_server_ros` with the `ros_to_rgbd` nodes.
76 5. In any test, the `rgbd_test_client` can be replaced by the combination of the `rgbd_to_ros` and
77 the `rgbd_test_client_ros` nodes.
