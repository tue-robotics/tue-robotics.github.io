22 BringApp and StartApp actions
23 ============================================================================
24 
25 The :class:`StartApp` and :class:`BringApp` action classes are used to
26 start an application and bring it to the foreground. :class:`StartApp`
27 starts an application by running an executable file, while
28 :class:`BringApp` first checks whether the application is already running
29 and if so brings it to the foreground, otherwise starts it by running the
30 executable file.
31 
32 
33 Example usage
34 ----------------------------------------------------------------------------
35 
36 The following example brings Notepad to the foreground if it is already
37 open, otherwise it starts Notepad::
38 
39  BringApp(r"C:\Windows\system32\\notepad.exe").execute()
40 
41 Note that the path to *notepad.exe* given above might not be correct for
42 your computer, since it depends on the operating system and its
43 configuration.
44 
45 In some cases an application might be accessible simply through the file
46 name of its executable, without specifying the directory. This depends on
47 the operating system's path configuration. For example, on the author's
48 computer the following command successfully starts Notepad::
49 
50  BringApp("notepad").execute()
51

66 class StartApp(ActionBase):
67  """
68  Start an application.
69 
70  When this action is executed, it runs a file (executable),
71  optionally with commandline arguments.
72 
73  """
74 
75  def __init__(self, *args, **kwargs):
76  """
77  Constructor arguments:
78  - *args* (variable argument list of *str*'s) --
79  these strings are passed to subprocess.Popen()
80  to start the application as a child process
81  - *cwd* (*str*, default *None*) --
82  if not *None*, then start the application in this
83  directory
84 
85  """
86  ActionBase.__init__(self)
87  self._args = args
88 
89  if "cwd" in kwargs: self._cwd = kwargs.pop("cwd")
90  else: self._cwd = None
91  if kwargs:
92  raise ArgumentError("Invalid keyword arguments: %r" % kwargs)
93 
94  # Expand any variables within path names.
95  self._args = [self._interpret(a) for a in self._args]
96  if self._cwd:
97  self._cwd = self._interpret(self._cwd)
98 
99  self._str = str(", ".join(repr(a) for a in self._args))
100 
101  def _interpret(self, path):
102  return os.path.expanduser(os.path.expandvars(path))
103 
104  def _execute(self, data=None):
105  self._log.debug("Starting app: %r" % (self._args,))
106  try:
107  Popen(self._args, cwd=self._cwd)
108  except Exception, e:
109  raise ActionError("Failed to start app %s: %s" % (self._str, e))

114 class BringApp(StartApp):
115  """
116  Bring an application to the foreground, starting it if it is not
117  yet running.
118 
119  When this action is executed, it looks for an existing window of
120  the application specified in the constructor arguments. If an
121  existing window is found, that window is brought to the
122  foreground. On the other hand, if no window is found the
123  application is started.
124 
125  Note that the constructor arguments are identical to those used by
126  the :class:`StartApp` action class.
127 
128  """
129 
130  def __init__(self, *args, **kwargs):
131  """
132  Constructor arguments:
133  - *args* (variable argument list of *str*'s) --
134  these strings are passed to :meth:`subprocess.Popen`
135  to start the application as a child process
136  - *cwd* (*str*, default *None*) --
137  if not *None*, then start the application in this
138  directory
139 
140  """
141  StartApp.__init__(self, *args, **kwargs)

144  self._log.debug("Bringing app: %r" % (self._args,))
145  target = self._args[0].lower()
146  focus_action = FocusWindow(executable=target)
