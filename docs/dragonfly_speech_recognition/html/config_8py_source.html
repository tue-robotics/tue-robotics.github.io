22 Configuration toolkit
23 ============================================================================
24 
25 Dragonfly's configuration toolkit makes it very easy to store
26 program data in a separate file from the main program logic. It uses
27 a three-phase *setup -- load -- use* system:
28 
29  - *setup* -- a Config object is created and its structure and
30  default contents are defined.
31  - *load* -- a separate file containing the user's configuration
32  settings is looked for and, if found, its values are loaded
33  into the Config object.
34  - *use* -- the program directly accesses the configuration
35  through easy Config object attributes.
36 
37 This configuration toolkit uses the following three classes:
38 
39  - :class:`Config` -- a collection of configuration settings, grouped
40  within one or more sections
41  - :class:`Section` -- a group of items and/or subsections
42  - :class:`Item` -- a single configuration setting
43 
44 Usage example
45 ----------------------------------------------------------------------------
46 
47 The main program using Dragonfly's configuration toolkit would
48 normally look something like this: ::
49 
50  from dragonfly import Config, Section, Item
51 
52  # *Setup* phase.
53  # This defines a configuration object with the name "Example
54  # configuration". It contains one section with the title
55  # "Test section", which has two configuration items. Both
56  # these items have a default value and a docstring.
57  config = Config("Example configuration")
58  config.test = Section("Test section")
59  config.test.fruit = Item("apple", doc="Must eat fruit.")
60  config.test.color = Item("blue", doc="The color of life.")
61 
62  # *Load* phase.
63  # This searches for a file with the same name as the main program,
64  # but with the extension ".py" replaced by ".txt". It is also
65  # possible to explicitly specify the configuration file's path.
66  # See Config.load() for more details.
67  config.load()
68 
69  # *Use* phase.
70  # The configuration values can now be accessed through the
71  # configuration object as follows.
72  print "The color of life is", config.test.color
73  print "You must eat an %s every day" % config.test.fruit
74 
75 The configuration defined above is basically complete. Every
76 configuration item has a default value and can be accessed by
77 the program. But if the user would like to modify some or all
78 of these settings, he can do so in an external configuration file
79 without modifying the main program code.
80 
81 This external configuration file is interpreted as Python code.
82 This gives its author powerful tools for determining the desired
83 configuration settings. However, it will usually consist merely
84 of variable assignments. The configuration file for the program
85 above might look something like this: ::
86 
87  # Test section
88  test.fruit = "banana" # Bananas have more potassium.
89  test.color = "white" # I like light colors.
90 
91 
92 Example command modules
93 ----------------------------------------------------------------------------
94 
95 The configuration toolkit is utilized in a number of command modules in the
96 *t4ngo/dragonfly-modules* repository, available on GitHub. See
97 :ref:`Related Resources: Command modules <RefCommandModulesList>`.
98 
99 
100 Implementation details
101 ----------------------------------------------------------------------------
102 
103 This configuration toolkit makes use of Python's special methods
104 for setting and retrieving object attributes. This makes it much
105 easier to use, as there is no need to use functions such as
106 *value = get_config_value("item_name")*; instead the configuration
107 values are immediately accessible as Python objects. It also allows
108 for more extensive error checking; it is for example trivial to
109 implement custom *Item* classes which only allow specific values
110 or value types, such as integers, boolean values, etc.

210  """
211  Load the configuration file at the given *path*, or
212  look for a configuration file associated with the calling
213  module.
214 
215  - *path* (*str*, default: *None*) --
216  path to the configuration file to load. If *None*,
217  then a path is generated from the calling module's
218  file name by replacing its extension with ".txt".
219 
220  If the *path* is a file, it is loaded. On the other hand,
221  if it does not exist or is not a file, nothing is loaded
222  and this configuration's defaults remain in place.
223 
224  """

226  if not path:

228  caller_file = caller_frame.f_globals["__file__"]
229  module_base, module_ext = os.path.splitext(caller_file)

324  def generate_config_file(self, path=None):
325  """
326  Create a configuration file containing this
327  configuration object's current settings.
328 
329  - *path* (*str*, default: *None*) --
330  path to the configuration file to load. If *None*,
331  then a path is generated from the calling module's
332  file name by replacing its extension with ".txt".
333 
334  """
335  # Config file header.
336  output = self._format_file_head()
337 
338  # Iterates through the sections.
339  stack = [iter(self._sections_list)]
340  names = []
341  while stack:
342  # Try to retrieve the next section from the top of the stack.
343  try:
344  section_name, section = next(stack[-1])
345  names.append(section_name)
346  except StopIteration:
347  # No more subsections, remove section from top of stack.
348  # Check for names needed for when the root Config is popped.
349  stack.pop()
350  if names: names.pop()
351  continue
352 
353  # Generate section head.
354  section_head = self._format_section_head(section_name, section, names)
355  output.extend(section_head)
356 
357  # Output section items.
358  for item_name, item in section._items_list:
359  item_output = self._format_item(item_name, item, names)
360  output.extend(item_output)
361 
362  # Push section onto the stack.
363  stack.append(iter(section._sections_list))
364 
365  if not path:
366  caller_frame = inspect.currentframe().f_back
367  caller_file = caller_frame.f_globals["__file__"]
368  module_base = os.path.splitext(caller_file)[0]
369  path = module_base + ".txt"
370 
371  f = open(path, "w")
372  f.write("\n".join(output))
373  f.close()
374  f = None

439 class Item(object):
440  """
441  Configuration item for storing configuration settings.
442 
443  Constructor arguments:
444  - *default* --
445  the default value for this item
446  - *doc* (*str*, default: *None*) --
447  an optional description of this item
448  - *namespace* (*dict*, default: *None*) --
449  an optional namespace dictionary which will be made available
450  to the Python code in the external configuration file
451  during loading
452 
453  A configuration item is the object that stores the actual
454  configuration settings. Each item has a default value, a current
455  value, an optional description, and an optional namespace.
456 
457  This class performs the checking of configuration values assigned
458  to it during loading of the configuration file. The default
459  behavior of this class is to only accept values of the same Python
460  type as the item's default value. So, if the default value is a
461  string, then the value assigned in the configuration file must
462  also be a string. Otherwise an exception will be raised and
463  loading will fail.
464 
465  Developers who want other kinds of value checking should override
466  the :meth:`Item.validate` method of this class.
467 
468  """
