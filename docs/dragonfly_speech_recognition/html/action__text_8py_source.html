24 """
25 Text action
26 ============================================================================
27 
28 This section describes the :class:`Text` action object. This type of
29 action is used for typing text into the foreground application. This works
30 on Windows, Mac OS and with X11 (e.g. on Linux).
31 
32 To use this class on X11/Linux, the
33 `xdotool <https://www.semicomplete.com/projects/xdotool/>`__ program must be
34 installed and the ``DISPLAY`` environment variable set. This class does
35 **not** support typing text in Wayland sessions.
36 
37 It differs from the :class:`Key` action in that :class:`Text` is used for
38 typing literal text, while :class:`dragonfly.actions.action_key.Key`
39 emulates pressing keys on the keyboard. An example of this is that the
40 arrow-keys are not part of a text and so cannot be typed using the
41 :class:`Text` action, but can be sent by the
42 :class:`dragonfly.actions.action_key.Key` action.
43 
44 
45 Windows Unicode Keyboard Support
46 ............................................................................
47 
48 The :class:`Text` action can be used to type arbitrary Unicode characters
49 using the `relevant Windows API <https://docs.microsoft.com/en-us/windows/desktop/api/winuser/ns-winuser-tagkeybdinput#remarks>`__.
50 This is disabled by default because it ignores the up/down status of
51 modifier keys (e.g. ctrl).
52 
53 It can be enabled by changing the ``unicode_keyboard`` setting in
54 `~/.dragonfly2-speech/settings.cfg` to ``True``::
55 
56  unicode_keyboard = True
57 
58 If you need to simulate typing arbitrary Unicode characters *and* have
59 *individual* :class:`Text` actions respect modifier keys normally for normal
60 characters, set the configuration as above and use the ``use_hardware``
61 parameter for :class:`Text` as follows:
62 
63 .. code:: python
64 
65  action = Text("σμ") + Key("ctrl:down") + Text("]", use_hardware=True) + Key("ctrl:up")
66  action.execute()
67 
68 Some applications require hardware emulation versus Unicode keyboard
69 emulation. If you use such applications, add their executable names to the
70 ``hardware_apps`` list in the configuration file mentioned above to make
71 dragonfly always use hardware emulation for them.
72 
73 If hardware emulation is required, then the action will use the keyboard
74 layout of the foreground window when calculating keyboard events. If any of
75 the specified characters are not typeable using the current window's
76 keyboard layout, then an error will be logged and no keys will be typed::
77 
78  action.exec (ERROR): Execution failed: Keyboard interface cannot type this character: 'μ'
79 
80 Keys in ranges 0-9, a-z and A-Z are always typeable. If keys in these ranges
81 cannot be typed using the current keyboard layout, then the equivalent key
82 will be used instead. For example, the following code will result in the 'я'
83 key being pressed when using the main Cyrillic keyboard layout: ::
84 
85  # This is equivalent to Text(u"яЯ").
86  Text("zZ").execute()
87 
88 These settings and parameters have no effect on other platforms.
89 
90 
91 X11/Linux Unicode Keyboard Support
92 ............................................................................
93 
94 The :class:`Text` action can also type arbitrary Unicode characters on X11.
95 This works regardless of the ``use_hardware`` parameter or
96 ``unicode_keyboard`` setting.
97 
98 Unlike on Windows, modifier keys will be respected by :class:`Text` actions
99 on X11. As such, the previous Windows example will work and can even be
100 simplified a little:
101 
102 .. code:: python
103 
104  action = Text("σμ") + Key("ctrl:down") + Text("]") + Key("ctrl:up")
105  action.execute()
106 
107 
108 It can also be done with one :class:`Key` action:
109 
110 .. code:: python
111 
112  Key("σ,μ,c-]").execute()

133 class Text(BaseKeyboardAction):
134  """
135  `Action` that sends keyboard events to type text.
136 
137  Arguments:
138  - *spec* (*str*) -- the text to type
139  - *static* (boolean) --
140  if *True*, do not dynamically interpret *spec*
141  when executing this action
142  - *pause* (*float*) --
143  the time to pause between each keystroke, given
144  in seconds
145  - *autofmt* (boolean) --
146  if *True*, attempt to format the text with correct
147  spacing and capitalization. This is done by first mimicking
148  a word recognition and then analyzing its spacing and
149  capitalization and applying the same formatting to the text.
150  - *use_hardware* (boolean) --
151  if *True*, send keyboard events using hardware emulation instead of
152  as Unicode text. This will respect the up/down status of modifier
153  keys.
154  """
155 
156  _specials = {
157  "\n": "enter",
158  "\t": "tab",
159  }
160 
161  def __init__(self, spec=None, static=False, pause=None,
162  autofmt=False, use_hardware=False):
163  if isinstance(spec, binary_type):
164  spec = spec.decode(getpreferredencoding())
165  BaseKeyboardAction.__init__(self, spec=spec, static=static,
166  use_hardware=use_hardware)
167  # Set other members.
168  self._autofmt = autofmt
169  self._pause = self._pause_default if pause is None else pause
170 
171  def _parse_spec(self, spec):
172  """Convert the given *spec* to keyboard events."""
173  key_symbols = []
174  for character in spec:
175  # The character is the key symbol, except in special cases.
176  if character in self._specials:
177  key_symbol = self._specials[character]
178  else:
179  key_symbol = character
180 
181  # Add the key symbol to the list.
182  key_symbols.append(key_symbol)
183 
184  return key_symbols
185 
186  def _execute_events(self, events):
187  """
188  Send keyboard events.
189 
190  If instance was initialized with *autofmt* True,
191  then this method will mimic a word recognition
192  and analyze its formatting so as to autoformat
193  the text's spacing and capitalization before
194  sending it as keyboard events.
195  """
196  if self._autofmt:
197  # Mimic a word, select and copy it to retrieve capitalization.
198  get_engine().mimic("test")
199  Key("cs-left, c-c/5").execute()
200  word = Clipboard.get_system_text()
201 
202  # Inspect formatting of the mimicked word.
203  index = word.find("test")
204  if index == -1:
205  index = word.find("Test")
206  capitalize = True
207  if index == -1:
208  self._log.error("Failed to autoformat.")
209  return False
210  else:
211  capitalize = False
212 
213  # Capitalize given text if necessary.
214  text = self._spec
215  if capitalize:
216  text = text[0].capitalize() + text[1:]
217 
218  # Reconstruct autoformatted output and convert it
219  # to keyboard events.
220  prefix = word[:index]
221  suffix = word[index + 4:]
222  events = self._parse_spec(prefix + text + suffix)
223 
224  # Calculate keyboard events.
225  use_hardware = self.require_hardware_events()
226  keyboard_events = []
227  for key_symbol in events:
228  # Get a Typeable object for each key symbol, if possible.
229  typeable = self._get_typeable(key_symbol, use_hardware)
230 
231  # Raise an error message if a Typeable could not be retrieved.
232  if typeable is None:
233  error_message = ("Keyboard interface cannot type this"
234  " character: %r" % key_symbol)
235  raise ActionError(error_message)
236 
237  # Get keyboard events using the Typeable.
238  keyboard_events.extend(typeable.events(self._pause))
239 
240  # Send keyboard events.
241  self._keyboard.send_keyboard_events(keyboard_events)
242  return True

245  return u"{!r}".format(self._spec)
