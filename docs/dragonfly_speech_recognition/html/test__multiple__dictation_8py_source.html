40 Mixed literal and dictation elements
41 -------------------------------------------------------------------------------
42 
43 Here we will investigate mixed, i.e. interspersed, fixed literal command
44 words and dynamic dictation elements. We will use the "MixedDictationRule"
45 class which has a spec of
46 "mixed [<dictation1>] <dictation2> command <dictation3>".
47 
48 Note that "<dictation1>" was made optional instead of "<dictation2>"
49 because otherwise the first dictation elements would always gobble up
50 all dictated words. There would (by definition) be no way to distinguish
51 which words correspond with which dictation elements. Such consecutive
52 dictation elements should for that reason be avoided in real command
53 grammars. The way the spec is defined now, adds some interesting
54 dynamics, because of the order in which they dictation elements parse
55 the recognized words. However, do note that that order is well defined
56 but arbitrarily chosen.
57

90 Repetition of dictation elements
91 -------------------------------------------------------------------------------
92 
93 Now let's take a look at repetition of dictation elements. For this
94 we will use the "RepeatedDictationRule" class, which defines its spec
95 as a repetition of "command <dictation>". I.e. "command Alice" will
96 match, and "command Alice command Bob" will also match.
97 
98 Note that this rule is inherently ambiguous, given the lack of a
99 clear definition of grouping or precedence rules for fixed literal
100 words in commands, and dynamic dictation elements. As an example,
101 "command Alice command Bob" could either match 2 repetitions with
102 "Alice" and "Bob" as dictation values, or a single repetition with
103 "Alice command Bob" as its only dictation value. The tests below
104 the show which of these actually occurs.
105
