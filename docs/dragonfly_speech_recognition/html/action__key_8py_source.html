25 Key action
26 ============================================================================
27 
28 This section describes the :class:`Key` action object. This
29 type of action is used for sending keystrokes to the foreground
30 application. This works on Windows, Mac OS and with X11 (e.g. on Linux).
31 Examples of how to use this class are given in :ref:`RefKeySpecExamples`.
32 
33 To use this class on X11/Linux, the
34 `xdotool <https://www.semicomplete.com/projects/xdotool/>`__ program must be
35 installed and the ``DISPLAY`` environment variable set. This class does
36 **not** support typing keys in Wayland sessions.
37 
38 .. _RefKeySpec:
39 
40 Keystroke specification format
41 ............................................................................
42 
43 The *spec* argument passed to the :class:`Key` constructor specifies which
44 keystroke events will be emulated. It is a string consisting of one or
45 more comma-separated keystroke elements. Each of these elements has one
46 of the following two possible formats:
47 
48 Normal press-release key action, optionally repeated several times:
49  [*modifiers* ``-``] *keyname* [``/`` *innerpause*] [``:`` *repeat*] [``/`` *outerpause*]
50 
51 Press-and-hold a key, or release a held-down key:
52  [*modifiers* ``-``] *keyname* ``:`` *direction* [``/`` *outerpause*]
53 
54 The different parts of the keystroke specification are as follows. Note
55 that only *keyname* is required; the other fields are optional.
56 
57  - *modifiers* --
58  Modifiers for this keystroke. These keys are held down
59  while pressing the main keystroke.
60  Can be zero or more of the following:
61 
62  - ``a`` -- alt key
63  - ``c`` -- control key
64  - ``s`` -- shift key
65  - ``w`` -- Windows key
66 
67  - *keyname* --
68  Name of the keystroke. Valid names are listed in
69  :ref:`RefKeySpecNames`.
70  - *innerpause* --
71  The time to pause between repetitions of this keystroke.
72  It should be given in hundredths of a second. For
73  example, "20" will pause for 20/100s = 0.2 seconds.
74  - *repeat* --
75  The number of times this keystroke should be repeated.
76  If not specified, the key will be pressed and released once.
77  - *outerpause* --
78  The time to pause after this keystroke.
79  It should be given in hundredths of a second. For
80  example, "20" will pause for 20/100s = 0.2 seconds.
81  - *direction* --
82  Whether to press-and-hold or release the key. Must be
83  one of the following:
84 
85  - ``down`` -- press and hold the key
86  - ``up`` -- release the key
87 
88  Note that releasing a key which is *not* being held down does *not*
89  cause an error. It harmlessly does nothing.

164 Example key actions
165 ............................................................................
166 
167 The following code types the text "Hello world!" into the foreground
168 application: ::
169 
170  Key("H, e, l, l, o, space, w, o, r, l, d, exclamation").execute()
171 
172 The following code is a bit more useful, as it saves the current file with
173 the name "dragonfly.txt" (this works for many English-language
174 applications): ::
175 
176  action = Key("a-f, a/50") + Text("dragonfly.txt") + Key("enter")
177  action.execute()
178 
179 The following code selects the next four lines by holding down the *shift*
180 key, slowly moving down 4 lines, and then releasing the *shift* key: ::
181 
182  Key("shift:down, down/25:4, shift:up").execute()
183 
184 The following code locks the screen by pressing the *Windows* key together
185 with the *l* key: ::
186 
187  Key("w-l").execute()

189 
190 Windows key support
191 ............................................................................
192 
193 Keyboard events sent by :class:`Key` actions on Windows are calculated using
194 the current foreground window's keyboard layout. The class will fallback on
195 Unicode events for keys not typeable with the current layout.
196 
197 The :class:`Key` action can be used to type arbitrary Unicode characters on
198 Windows using the `relevant Windows API
199 <https://docs.microsoft.com/en-us/windows/desktop/api/winuser/ns-winuser-tagkeybdinput#remarks>`__.
200 This is disabled by default because it ignores the up/down status of
201 modifier keys (e.g. ctrl).
202 
203 It can be enabled by changing the ``unicode_keyboard`` setting in
204 `~/.dragonfly2-speech/settings.cfg` to ``True``: ::
205 
206  unicode_keyboard = True
207 
208 The ``use_hardware`` parameter can be set to ``True`` if you need to
209 selectively require hardware events for a :class:`Key` action: ::
210 
211  # Passing use_hardware=True will guarantee that Ctrl+C is always
212  # pressed, regardless of the layout. See below.
213  Key("c-c", use_hardware=True).execute()
214 
215 If the Unicode keyboard is not enabled or the ``use_hardware`` parameter is
216 ``True``, then no keys will be typed and an error will be logged for
217 untypeable keys: ::
218 
219  action.exec (ERROR): Execution failed: Keyboard interface cannot type this character: 'μ'
220 
221 Keys in ranges 0-9, a-z and A-Z are always typeable. If keys in these ranges
222 cannot be typed using the current keyboard layout, then the equivalent key
223 will be used instead. For example, the following code will result in the 'я'
224 key being pressed when using the main Cyrillic keyboard layout: ::
225 
226  # This is equivalent to Key(u"я, Я, c-я").
227  Key("z, Z, c-z", use_hardware=True).execute()
228 
229 Unlike the :class:`Text` action, individual :class:`Key` actions can send
230 both hardware *and* Unicode events. So the following example will work if
231 the Unicode keyboard is enabled: ::
232 
233  # Type 'σμ' and then press Ctrl+Z.
234  Key(u"σ, μ, c-z").execute()
235 
236 
237 X11 key support
238 ............................................................................
239 
240 The :class:`Key` action can be used to type arbitrary keys and Unicode
241 characters on X11/Linux. It is not limited to the key names listed above,
242 although all of them will work too.
243 
244 Unicode characters are supported on X11 by passing their Unicode code point
245 to the keyboard implementation. For example, the character ``'€'`` is
246 converted to ``'U20AC'``. The Unicode code point can also be passed
247 directly, e.g. with ``Key('U20AC')``.
248 
249 Unlike on Windows, the :class:`Key` action is able to use modifiers with
250 Unicode characters on X11.
251 
252 This class does **not** support typing keys in Wayland sessions.
253

255 Example X11 key actions
256 ----------------------------------------------------------------------------
257 
258 In addition to the examples in the previous section, the following example
259 will work on X11/Linux.
260 
261 The following code will type 'σμ' into the foreground application and then
262 press ctrl+z: ::
263 
264  Key("σ,μ,c-z").execute()
265 
266 The following code will press 'µ' while holding control and alt: ::
267 
268  Key("ca-μ").execute()

271 
272  Key("XF86Refresh").execute()

294 class Key(BaseKeyboardAction):
295  """
296  Keystroke emulation action.
297 
298  Constructor arguments:
299  - *spec* (*str*) -- keystroke specification
300  - *static* (boolean) -- flag indicating whether the
301  specification contains dynamic elements
302  - *use_hardware* (boolean) --
303  if *True*, send keyboard events using hardware emulation instead
304  of as Unicode text. This will respect the up/down status of
305  modifier keys.
306 
307  The format of the keystroke specification *spec* is described in
308  :ref:`RefKeySpec`.
309 
310  This class emulates keyboard activity by sending keystrokes to the
311  foreground application. It does this using Dragonfly's keyboard
312  interface for the current platform. The implementation for Windows
313  uses the ``sendinput()`` Win32 API function. The implementation
314  for Mac OS uses
315  `pynput <https://pynput.readthedocs.io/en/latest/>`__. The
316  implementation for X11/Linux uses `xdotool
317  <https://www.semicomplete.com/projects/xdotool/>`__.
318 
319  """
320 
321  # Various keystroke specification format parameters.
322  _key_separator = ","
323  _delimiter_characters = ":/"
324  _modifier_prefix_delimiter = "-"
325  _modifier_prefix_characters = {
326  'a': typeables["alt"],
327  'c': typeables["control"],
328  's': typeables["shift"],
329  'w': typeables["win"],
330  }
331  interval_factor = 0.01
332  interval_default = 0.0
333 
334  # Keystroke event data type.
335  EventData = collections.namedtuple(
336  "EventData", "keyname "
337  "direction "
338  "modifiers "
339  "inner_pause "
340  "repeat "
341  "outer_pause "

344  def _parse_spec(self, spec):
345  # Iterate through the keystrokes specified in spec, parsing
346  # each individually.
347  events = []
348  for single in spec.split(self._key_separator):
349  event_data = self._parse_single(single)
350  if not event_data:
351  continue
352  events.append(event_data)
353  return events
354 
355  def _parse_single(self, spec):
356  # pylint: disable=R0912,R0914,R0915
357  # Suppress warnings about too many branches, variables and
358  # statements.
359 
360  # Remove leading and trailing whitespace.
361  spec = spec.strip()
362  if not spec:
363  return None
364 
365  # Parse modifier prefix.
366  index = spec.find(self._modifier_prefix_delimiter)
367  if index != -1:
368  s = spec[:index]
369  index += 1
370  modifiers = []
371  for c in s:
372  if c not in self._modifier_prefix_characters:
373  raise ActionError("Invalid modifier"
374  " prefix character: %r" % c)
375  m = self._modifier_prefix_characters[c]
376  if m in modifiers:
377  raise ActionError("Double modifier"
378  " prefix character: %r" % c)
379  modifiers.append(m)
380  else:
381  index = 0
382  modifiers = ()
383 
384  inner_pause = None
385  special = None
386  outer_pause = None
387 
388  delimiters = [(c, i + index) for i, c in enumerate(spec[index:])
389  if c in self._delimiter_characters]
390  delimiter_sequence = "".join([d[0] for d in delimiters])
391  delimiter_index = [d[1] for d in delimiters]
392 
393  if delimiter_sequence == "":
394  keyname = spec[index:]
395  elif delimiter_sequence == "/":
396  keyname = spec[index:delimiter_index[0]]
397  outer_pause = spec[delimiter_index[0]+1:]
398  elif delimiter_sequence == "/:":
399  keyname = spec[index:delimiter_index[0]]
400  inner_pause = spec[delimiter_index[0]+1:delimiter_index[1]]
401  special = spec[delimiter_index[1]+1:]
402  elif delimiter_sequence == "/:/":
403  keyname = spec[index:delimiter_index[0]]
404  inner_pause = spec[delimiter_index[0]+1:delimiter_index[1]]
405  special = spec[delimiter_index[1]+1:delimiter_index[2]]
406  outer_pause = spec[delimiter_index[2]+1:]
407  elif delimiter_sequence == ":":
408  keyname = spec[index:delimiter_index[0]]
409  special = spec[delimiter_index[0]+1:]
410  elif delimiter_sequence == ":/":
411  keyname = spec[index:delimiter_index[0]]
412  special = spec[delimiter_index[0]+1:delimiter_index[1]]
413  outer_pause = spec[delimiter_index[1]+1:]
414  else:
415  raise ActionError("Invalid key spec: %s" % spec)
416 
417  if inner_pause is not None:
418  s = inner_pause
419  try:
420  inner_pause = float(s) * self.interval_factor
421  if inner_pause < 0: raise ValueError
422  except ValueError:
423  raise ActionError("Invalid inner pause value: %r,"
424  " should be a positive number." % s)
425  if outer_pause is not None:
426  s = outer_pause
427  try:
428  outer_pause = float(s) * self.interval_factor
429  if outer_pause < 0: raise ValueError
430  except ValueError:
431  raise ActionError("Invalid outer pause value: %r,"
432  " should be a positive number." % s)
433  else:
434  outer_pause = self.interval_default * self.interval_factor
435 
436  direction = None; repeat = 1
437  if special is not None:
438  if special == "down": direction = True
439  elif special == "up": direction = False
440  else:
441  s = special
442  try:
443  repeat = int(s)
444  if repeat < 0: raise ValueError
445  except ValueError:
446  raise ActionError("Invalid repeat value: %r,"
447  " should be a positive integer." % s)
448 
449  if direction is not None:
450  if modifiers:
451  raise ActionError("Cannot use direction with modifiers.")
452  if inner_pause is not None:
453  raise ActionError("Cannot use direction with inner pause.")
454 
455  return self.EventData(keyname, direction, modifiers, inner_pause,
456  repeat, outer_pause)
457 
458  def _execute_events(self, events):
459  # Calculate keyboard events from events (event data).
460  use_hardware = self.require_hardware_events()
461  keyboard_events = []
462  for event_data in events:
463  events_single = self._calc_events_single(event_data,
464  use_hardware)
465  keyboard_events.extend(events_single)
466 
467  # Send keyboard events.
468  self._keyboard.send_keyboard_events(keyboard_events)
469  return True
470 
471  def _calc_events_single(self, event_data, use_hardware):
472  (keyname, direction, modifiers, inner_pause, repeat,
473  outer_pause) = event_data
474 
475  # Get a Typeable object for the key, if possible.
476  typeable = self._get_typeable(event_data.keyname, use_hardware)
477 
478  # Raise an error message if a Typeable could not be retrieved.
479  if typeable is None:
480  error_message = ("Keyboard interface cannot type this "
481  "character: %r (in %r)" %
482  (keyname, self._spec))
483  raise ActionError(error_message)
484 
485  # Calculate keyboard events using the Typeable and event data.
486  if direction is None:
487  if inner_pause is None:
488  inner_pause = self.interval_default * self.interval_factor
489  if repeat == 0:
490  events = []
491  else:
492  events = []
493  for m in modifiers:
494  events.extend(m.on_events())
495  for _ in range(repeat - 1):
496  events.extend(typeable.events(inner_pause))
497  events.extend(typeable.events(outer_pause))
498  for m in modifiers[-1::-1]:
499  events.extend(m.off_events())
500  else:
501  if direction:
502  events = typeable.on_events(outer_pause)
503  else:
504  events = typeable.off_events(outer_pause)
505  return events
